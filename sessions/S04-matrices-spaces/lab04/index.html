Lab 04 — Matrices in Unity: order, pivots & inverses

Week 4 · Thursday in-person studio · Dr. Marcel Gavriliu · bring your laptop

Tuesday's lecture packaged translate, rotate, and scale into one 4×4 matrix, composed by multiplication (read right-to-left), undone by an inverse, and steered onto any pivot with the sandwich T(p)·R·T(-p). Today you run the real Topic4 projects that do exactly this and poke them — each project uploads a hand-built matrix to the shader as MyXformMat. The studio is predict-then-run: write your prediction down before you press Play, then reconcile. The instructor circulates. The second half is MP3 lab time.

Predict before you run — every time. The whole point of the studio is to catch the moment your mental model and the screen disagree. If you press Play first, you have learned nothing. Say the answer out loud or jot it, then run.

1 · TSvsST — order is the operation

Goal. Watch the same scale and translate produce two different results depending on multiplication order. XformLoader has a mode field: ScaleThenTranslate builds tm * sm, TranslateThenScale builds sm * tm.

Where to look: 4.5.TSvsST/Assets/Source/Model/XformLoader.cs — the switch(mode) block. Flip mode in the Inspector on the object that has the script; adjust the object's Scale and Position in the Transform to make the effect large.
CHECKPOINT 1a

Predict which cube moves where

Set a non-uniform scale (say 2, 1, 1) and a clear offset (say 3, 0, 0).

Predict first: in ScaleThenTranslate (tm * sm, scale acts first), does the offset get stretched, or stay 3? In TranslateThenScale (sm * tm, translate acts first), the scale multiplies the whole matrix — what happens to the 3 then? Write both predicted x-positions before you flip mode.
CHECKPOINT 1b

Read it off the matrix column

Column 3 of MyXformMat is the translation (lecture Part 1). Reason about column 3 for each order.

Predict first: for sm * tm with scale (2,1,1) and translate (3,0,0), predict column 3 by hand — does the x entry read 3 or 6? Match your prediction to where the cube lands. Which order is the one Unity's own Matrix4x4.TRS uses (scale first, then rotate, then translate)?

2 · PivotedScaleRotate — the pivot sandwich

Goal. Rotate an object about a chosen pivot instead of its own origin. XformLoader builds m = pm * m * ipm = T(p)·M·T(-p), with the pivot exposed as PivotPosition (an offset in the object's own space).

Where to look: 4.7.PivotedScaleRotate/Assets/Source/Model/XformLoader.cs — the pm * m * ipm line and the PivotPosition field. A small marker (ShowPivotPosition) is drawn at the pivot. Toggle ShowRotation on to see it spin.
CHECKPOINT 2a

Predict the fixed point

Set PivotPosition to a clear offset — try (1.5, 0, 0).

Predict first: lecture said the pivot point itself is the fixed point of T(p)·R·T(-p) — it does not move. Before enabling rotation, predict: as the object spins, which single point on (or near) it stays put? Point at that spot on screen, then turn ShowRotation on and check the marker.
CHECKPOINT 2b

Move the pivot, re-predict the arc

Change PivotPosition to (0, 0, 0), then to a large offset like (3, 0, 0).

Predict first: at pivot (0,0,0) the object spins in place; at a large offset it sweeps a wide arc. Predict the radius of the swept circle for pivot (3,0,0) — what distance is the object from its own pivot? Then predict: is this T·R behavior or R·T behavior from lecture Part 2, and why?
CHECKPOINT 2c

Where did column 3 go?

Lecture noted the translation column of a pivoted rotation is (I - R)p — not the pivot, not zero.

Predict first: without computing exactly, predict whether column 3 of the final MyXformMat equals PivotPosition. (It does not — that is why you build the sandwich instead of typing a translation by hand.) If you had to hand-enter a pivoted rotation as a single TRS, what would go wrong?

3 · InverseTransform — undo, and change frames

Goal. Watch M⁻¹ peel a transform apart. XformLoader builds invT, invR, invS separately and chains invS · invR · invT onto a copy of a reference tree; a DeltaPosition slider slides the inverted copies apart so you can see each stage.

Where to look: 4.6.InverseTransform/Assets/Source/Model/XformLoader.cs — the ToShow enum (ShowInvT, ShowInvRT, ShowInvSRT, ShowTRS) and the three inverse constructions. Cycle ToShow and drag DeltaPosition.
CHECKPOINT 3a

Predict what each inverse undoes

Cycle ToShow from ShowTRS through ShowInvT, ShowInvRT, ShowInvSRT.

Predict first: the reversal rule is (T·R·S)⁻¹ = S⁻¹·R⁻¹·T⁻¹ — undo in reverse order. Predict what invT · TRS leaves behind (translation gone, rotation + scale remain?), then invR · invT · TRS, then the full invS · invR · invT · TRS. What pose should the last one land on?
CHECKPOINT 3b

Rotation inverse is a transpose

The code inverts rotation with Quaternion.AngleAxis(-rotAngle, rotAxis) — same axis, negated angle.

Predict first: lecture said a rotation matrix is orthonormal, so R⁻¹ = Rᴸ (its transpose) — no division, no new trig. The code negates the angle instead; convince yourself on paper these give the same matrix. Then predict: what does the "assuming no zeros" comment on the scale inverse guard against, and what would invS do if a scale factor were 0?
